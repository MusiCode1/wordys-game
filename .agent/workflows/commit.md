---
description: תהליך ביצוע קומיט מסודר לגיט
---

1. וודא שאין שגיאות בפרויקט באמצעות הרצת הפקודה `npm run check` או בדיקת הפלט של `/watch-check`. וודא שהפקודה הסתיימה בהצלחה לפני המעבר לשלב הבא.
2. עדכן את דו"ח העבודה (Walkthrough) בשינויים שבוצעו באמצעות `/update_walkthrough`.
3. הרץ `git status` כדי לראות את מצב הקבצים. בדוק שהפלט תקין.
4. אם יש צורך, הרץ `git diff` כדי לבדוק את השינויים.
5. **אין לשגר פקודות בשרשרת (&&) או מספר פקודות במקביל.** בצע כל פקודת git בנפרד ובדוק שהיא בוצעה בהצלחה (Status: DONE).
6. הוסף ל-Stage **רק** את הקבצים הרלוונטיים לקומיט הנוכחי באמצעות `git add <files>`. בדוק שהפקודה הצליחה.
7. נסח הודעת קומיט ברורה ותמציתית **בשפה העברית בלבד**.
8. בקש את אישור המשתמש להודעת הקומיט ולרשימת הקבצים.
9. לאחר האישור, בצע את הקומיט באמצעות `git commit -m "ההודעה בעברית"`. וודא שהקומיט נוצר בהצלחה.
